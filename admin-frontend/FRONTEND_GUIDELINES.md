# Руководство по Фронтенду

## Описание
Данные правила применимы к клиентской части приложения (платформа для учеников) и админ-панели. Основная цель: снизить нагрузку на сервер и обеспечить гладкий, безлаговый опыт пользователям даже при плохом интернете.

## Критические правила деплоя и работы интерфейса

1. **Хостинг интерфейса (Деплой):**
   - Фронтенд (React, Vue, Vite, SPA) **НЕ деплоится** на Railway вместе с бэкендом.
   - Интерфейс выносится на специализированные платформы: **Vercel** или **Cloudflare Pages**. Это обеспечивает бесплатную, мгновенную раздачу файлов через мировую CDN сеть и полностью снимает нагрузку с вашего бэкенда на Railway.

2. **Проигрывание видео-материалов:**
   - На фронтенде используются видео-плееры с обязательной поддержкой адаптивного потока — **HLS** (`.m3u8`). (Примеры библиотек: `video.js`, `hls.js`, кастомные UI плееры).
   - Плеер запрашивает фрагменты видео напрямую из стороннего CDN-хостинга для видео (Cloudflare Stream/Mux), полностью минуя ваш Railway API и Supabase.

3. **Оптимизация запросов тестирования:**
   - Логика отправки ответов на вопросы должна быть надежной (например, отправлять по одному или батчами, поддерживать кэширование при кратковременном обрыве интернета у ученика).
   - При массовом онлайне фронтенд должен избегать повторных ненужных запросов (использовать кэширование типа React Query / SWR / Zustand).
